# Checkbox Component

A versatile checkbox component that supports both standard checkbox and switch variants. The component is highly customizable with various sizes, colors, and states, making it suitable for a wide range of use cases.

## Features

- Two variants: standard checkbox and switch
- Multiple sizes (small, medium, large)
- Customizable colors
- Support for indeterminate state
- Label positioning options
- Helper text and error message support
- Fully accessible
- RTL support
- Keyboard navigation

## Usage

```tsx
import { Checkbox } from './components/basic/Checkbox';

// Basic checkbox
<Checkbox label="Accept terms" />

// Switch variant
<Checkbox variant="switch" label="Enable notifications" />

// With helper text
<Checkbox
  label="Subscribe to newsletter"
  helperText="You will receive weekly updates"
/>

// With error state
<Checkbox
  label="Required field"
  error="This field is required"
/>

// Indeterminate state
<Checkbox
  label="Select all"
  indeterminate={true}
/>
```

## Best Practices

1. **Labels**
   - Always provide clear and concise labels
   - Use action-oriented language for better user understanding
   - Position labels consistently across your application

2. **States**
   - Use the indeterminate state when some, but not all, items in a group are selected
   - Clearly indicate disabled states with reduced opacity
   - Show error states when validation fails

3. **Accessibility**
   - Ensure proper ARIA attributes are maintained
   - Maintain keyboard navigation support
   - Use sufficient color contrast for visibility
   - Provide meaningful error messages

4. **Layout**
   - Maintain consistent spacing between checkbox groups
   - Align checkboxes properly with other form elements
   - Consider mobile responsiveness when positioning labels

## CSS Customization

The Checkbox component can be customized using CSS modules. Here are the main classes available:

```css
.container /* Main container */
.inputWrapper /* Wrapper for the input element */
.input /* Hidden native input */
.checkbox /* Custom checkbox style */
.switch /* Switch variant style */
.label /* Label style */
.helperText /* Helper text style */
.error /* Error message style */
```

You can override these classes by providing a custom className prop:

```tsx
<Checkbox
  className="custom-checkbox"
  label="Custom styled checkbox"
/>
```
