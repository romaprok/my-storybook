# Input Component

A versatile input component that supports various styles, states, and features.

## Features

- Multiple variants (outlined, filled, standard)
- Different sizes (small, medium, large)
- Label and placeholder support
- Helper text and error states
- Icon support (start and end icons)
- Clearable option
- Full width support
- Disabled and readonly states
- Required field indication
- RTL support
- Accessibility features

## Usage

```tsx
import { Input } from './Input';

// Basic usage
<Input placeholder="Basic input" />

// With label and helper text
<Input
  label="Email"
  helperText="We'll never share your email"
  placeholder="Enter your email"
/>

// With error state
<Input
  label="Username"
  error="This username is already taken"
  value="existing-user"
/>

// With icons
<Input
  startIcon="🔍"
  endIcon="✓"
  placeholder="Search..."
/>
```

## Accessibility

The Input component follows accessibility best practices:

- Uses semantic HTML elements
- Supports keyboard navigation
- Provides proper labeling
- Includes ARIA attributes
- Communicates error states
- Supports screen readers

## Best Practices

1. **Labels and Placeholders**
   - Always use labels for form inputs
   - Use placeholders for additional hints, not as replacements for labels
   - Make labels clear and concise

2. **Error Handling**
   - Display error messages clearly
   - Use red color sparingly and with sufficient contrast
   - Provide helpful error messages that guide users to fix the issue

3. **Icons**
   - Use icons that are universally recognizable
   - Ensure icons have sufficient contrast
   - Add tooltips for icon buttons when necessary

4. **Validation**
   - Validate input in real-time when appropriate
   - Show validation feedback immediately after user interaction
   - Use helper text to prevent errors before they occur

5. **Responsive Design**
   - Use fullWidth prop for mobile views
   - Ensure touch targets are large enough
   - Test with different screen sizes
